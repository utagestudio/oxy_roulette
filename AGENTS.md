# プロジェクト名/指示書
Oxygen Not Included 配信用 ルーレットアプリ の作成

## 使用シーン
- 配信冒頭で「やることリスト」からランダム抽選を視覚的に見せる用途
- 毎配信、冒頭で1回だけ回す想定のルーレット

## 概要
- ブラウザで動くルーレットアプリを作る
  - ここでいう「ルーレット」とは「リストハイライト形式」を表し、便宜上「ルーレット」と呼称する
- ルーレットの項目は、ユーザーが任意に指定できる（抽選項目仕様参照）
- ルーレットのスタートボタンを配置し、クリックするとルーレットが回る
  - ルーレット中は、ランダムな項目が選択状態を表す表現になり、一定時間そのセルが切り替わり続ける
  - 停止する直前の切り替わりタイミングは少しずつ遅くなる
- 停止はランダムな一定時間で止まる
  - 総時間は `6.0秒〜10.0秒` のランダム
  - 更新間隔は `80ms` 開始、終盤で `220ms` まで増加（減速演出）
  - 最終当選項目は抽選開始時点で決定し、演出はその表示切替として扱う
- 想定解像度は `1600x900` 固定（ブラウザズーム100%前提）
- ルーレット項目および状態は、`localStorage` に保存し、再度利用する際には復元する

### 抽選項目仕様
- ルーレットの項目は、1行1項目のテキストで入力
  - 複数行のペーストにも対応
  - 入力・管理欄は、ルーレットとは別に用意する
- 入力仕様
  - 貼り付け・追加時は改行で分割して一括追加する
  - 各行は `trim` し、空行（空文字）は無視する
  - 新規追加項目の初期状態は `対象` とする
  - 重複項目（同一テキスト）は禁止する
- 各行は「非対象」「対象」「抽選済み」のどれかの状態を持つ
  - 対象：実際にルーレットを回したときに止まる可能性のある項目
  - 非対象：ルーレットに含まれない項目
  - 抽選済み：過去にルーレットで選ばれた項目（当選後、`Accept` ボタンで遷移）
- GUIで3状態を任意に切り替えることができる
- 抽選対象は `status === "target"` の項目のみ
- 対象が0件のときは抽選不可（`Start` 無効化 + メッセージ表示）

### 抽選確定ルール
- 抽選中は再 `Start` 不可
- 抽選終了後、`Accept` 前であれば再 `Start` 可能
- `Accept` が押された場合のみ当選結果を確定し、当選項目を `抽選済み` に変更する
- `Accept` しない限り、状態は確定しない（やり直し時は状態変更なし）

### ルーレット表示仕様
- 「円形ルーレット」ではなく、表組み（グリッド）表示にする
- グリッドは `target` 状態の項目数に応じた可変レイアウトとする
- 空セルは `Empty` として表示し、抽選対象外とする
- 各セルのテキストは折り返して表示する（必要に応じて改行して全体を視認しやすくする）

### 例外時の挙動
- 全件 `抽選済み` になった場合に備え、個別に状態を戻せるUIを必須化する

## ファイル構造（初期案）
- `src/main.tsx`：エントリポイント

- `src/hooks/useRoulette.ts`：抽選ロジック・状態遷移
- `src/utils/storage.ts`：`localStorage` 入出力
- `src/utils/random.ts`：乱数・抽選ユーティリティ
- `src/styles/`：スタイル

## localStorage 保存仕様
- 保存キー：`oni-roulette-v1`
- 保存データ：
  - `items: { id: string; text: string; status: "inactive" | "target" | "done" }[]`
  - `lastResultId: string | null`
  - `updatedAt: string`（ISO文字列）
- 起動時にJSONパース失敗時は初期化する

## ワークフロー
1. （ユーザー操作）ページを開く
2. （アプリ）初回でなければ、`localStorage` から前回データをロードして反映する
3. （ユーザー操作）初回であれば、ルーレット項目入力欄に項目を入力する
4. （ユーザー操作）項目管理UIで、今回の抽選対象（`target`）を調整する
5. （ユーザー操作）`Start` ボタンを押す
6. （アプリ）`target` のみを対象にルーレット抽選を実施する
7. （アプリ）抽選結果表示後、`Accept` ボタンを有効化する
8. （ユーザー操作）
  - 結果を確定する場合：`Accept` を押す
  - 確定しない場合：`Accept` は押さずに再 `Start` する
9. （アプリ）`Accept` が押された場合のみ、当選項目を `抽選済み` に変更する
10. （アプリ）次回抽選に向けて 4. 以降を繰り返す
