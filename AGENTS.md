# プロジェクト名/指示書
Oxygen Not Included 配信用 ルーレットアプリ の作成

## 概要
- ブラウザで動くルーレットアプリを作る
  - ここでいう「ルーレット」とは「リストハイライト形式」を表し、便宜上「ルーレット」と呼称する
- ルーレットの項目は、ユーザーが任意に指定できる（抽選項目仕様参照）
- ルーレットのスタートボタンを配置し、クリックするとルーレットが回る
  - ルーレット中は、ランダムな項目が選択状態を表す表現になり、一定時間そのセルが切り替わり続ける
  - 停止する直前の切り替わりタイミングは少しずつ遅くなる
- 停止はランダムな一定時間で止まる
- 想定解像度は 1600x900
- ルーレット項目および状態は、localStrage に保存し、再度利用する際には復元する

### 抽選項目仕様
- ルーレットの項目は、1行1項目のテキストで入力
  - 複数行のペーストにも対応
  - 入力・管理欄は、ルーレットとは別に用意する
- 各行は「非対象」「対象」「抽選済み」のどれかの状態を持つ
  - 対象： 実際にルーレットを回したときに止まる可能性のある項目
  - 非対象： のちのち追加することを想定した、ルーレットに含まれない項目
  - 抽選済み： 過去にルーレットで選ばれた項目（当選後、Acceptボタンを押すことで変化）
- 入力された項目の初期状態は「対象」にしておき、GUIで3状態を任意に切り替えることができる。
- ルーレット後、Acceptボタンを押すと、その項目が抽選済み状態に移行する。
- 「円形ルーレット」ではない、ルーレットビジュアルにしたい
  - n*mの表組み状にする

## ワークフロー
1. （ユーザー操作） ページを開く
2. （ユーザー操作） 初回であれば、ルーレット項目入力欄に項目を入力させる
3. （アプリ） 初回じゃなければ、localStrageから前回のデータをロードしてきて、反映させる
4. （ユーザー操作） ルーレット項目入力欄から、今回の抽選の対象となる項目を選択する
5. （ユーザー操作） スタートボタンを押す
6. （アプリ） ルーレットによる抽選を実施
7. （アプリ）抽選結果が出たら、Acceptボタンを有効化する
8. （ユーザー操作） 結果に問題がなければ、Acceptボタンを押す。問題があれば ワークフロー4からやり直す
9. （アプリ）Acceptボタンが押されたら、抽選された項目の状態を「抽選済み」に変更する
10. ワークフロー4から繰り返す
